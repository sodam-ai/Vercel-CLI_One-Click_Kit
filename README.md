# Vercel 원클릭 키트 (Vercel CLI One-Click Kit)

> 컴퓨터·명령어가 처음인 분도 **더블클릭 한 번**으로 Vercel을 설치하고,
> 내가 만든 웹사이트를 인터넷에 올릴(배포) 수 있게 도와주는 **한국어 도우미**입니다.

- **Vercel**: 내가 만든 웹사이트/앱을 인터넷에 무료로 올려주는(배포해주는) 서비스.
- **이 키트**: 그 일을 **검은 명령어 창 대신 한국어 메뉴**로 하게 해 줍니다.
- **관리자 권한(UAC) 없이** 설치됩니다. (Node.js 자동 설치도 무관리자 방식)

> 🇬🇧 English: **[README.en.md](README.en.md)** · 단계별 따라하기: **[왕초보가이드.md](왕초보가이드.md)**

---

## ⚡ 빠른 시작 (3단계, 이거 하나만 기억하세요)

| 단계 | 할 일 |
|---|---|
| **1** | 이 폴더에서 **`시작하기.bat`** 더블클릭 |
| **2** | 화면 맨 위 **">> 지금 할 일"** 줄에 적힌 번호 누르기 (보통 `[1] 설치` → `[2] 사용 → 로그인` → `배포`) |
| **3** | 끝! 화면이 매번 다음에 누를 번호를 알려줍니다 |

> 화살표 **↑ ↓** 로 고르고 **Enter**, 또는 **번호키**를 눌러도 됩니다.

## 📋 사전 준비물 / 필요 프로그램

| 항목 | 필요 여부 | 설명 |
|---|---|---|
| **Windows 10 / 11** | 필수 | 이 키트는 Windows 전용입니다. |
| **인터넷 연결** | 필수 | 설치·로그인·배포에 필요(와이파이/랜). |
| **Node.js 18 이상** | 자동 | 없으면 키트가 **무관리자 휴대용**으로 자동 설치(또는 nodejs.org 안내). |
| **Vercel CLI** | 자동 | `[1] 설치하기`가 자동으로 깝니다. |
| **인터넷 브라우저** | 로그인 시 | Vercel 로그인은 브라우저로 진행됩니다. |
| **Vercel 계정** | 로그인 시 | 없으면 브라우저에서 무료로 바로 만들 수 있어요(GitHub/이메일). |
| **관리자 권한** | ❌ 불필요 | 관리자 허용(UAC) 창이 뜨지 않습니다. |

## 📥 다운로드 / 설치 방법

1. **키트 받기**: 이 폴더(또는 zip)를 받았다면, zip은 **압축을 풀어** 한 폴더에 둡니다.
2. **차단 해제**: 인터넷에서 받은 파일은 Windows가 막을 수 있어요. → `시작하기.bat`이 **자동으로 차단을 풀어줍니다.** (안 되면: 파일 우클릭 → 속성 → 아래 **차단 해제** 체크 → 확인)
3. **설치**: `시작하기.bat` 더블클릭 → **`[1] 설치하기`**.
   - Node.js가 없으면 → **`Y`(자동, 무관리자)** 또는 nodejs.org 안내. 설치 후 **창을 닫고 다시 실행**.
   - 검은 설치 창이 영어로 떠도 정상입니다. **닫지 말고** 기다리세요(3~10분).

> 💡 파란 경고(SmartScreen)가 뜨면 → **추가 정보 → 실행**. (Windows가 새 파일을 조심하라고 띄우는 정상 경고)

## 🖥️ 실행 방법 / 처음 화면

`시작하기.bat`을 열면 맨 위에 **내 컴퓨터 상태**가 보입니다.

```
[지금 내 컴퓨터 상태]
 - Node.js (필수 부품)   : 있음  [OK]
 - Vercel CLI            : 없음  [설치 필요]
 - Vercel 로그인         : 확인 안 됨  [로그인 필요]

 >> 지금 할 일: [1] 설치하기 를 누르세요.
```

- **초록 [OK]** = 준비됨 / **노랑 [필요]** = 화면이 시키는 번호를 누르면 됩니다.

## 🧭 워크플로우 (전체 흐름)

```
시작하기.bat
   → [1] 설치하기            (처음 1번: Node.js + Vercel CLI)
   → [2] 사용하기 → 로그인    (처음 1번: 브라우저에서 로그인)
   → [2] 사용하기 → (연결)    (선택: 이 폴더를 Vercel과 연결)
   → [2] 사용하기 → 배포 → 미리보기(안전, 임시 주소)
   → 확인되면 → 진짜 공개(YES 입력)
```

> 막히면 언제든 **`[6] 내 상태 점검 / 문제 해결`**.

## 📂 사용 방법 — 시작하기 메뉴 (한국어)

| 번호 | 이름 | 하는 일 |
|---|---|---|
| 1 | **설치하기** | Vercel CLI 설치(Node.js 없으면 자동 안내). **이미 설치돼 있으면 바로 끝나요.** |
| 2 | **사용하기** | 로그인 · 연결 · **배포** · **고급 기능**(도메인/환경변수/로그/승격) |
| 3 | **제거하기** | 깨끗이 지움(내 코드·파일은 안 지움) |
| 4 | **사용설명서** | 왕초보 가이드 열기 |
| 5 | **대시보드 열기** | 인터넷의 Vercel 관리 페이지 |
| 6 | **내 상태 점검 / 문제 해결** | 상태 재확인 + 오류 해결 + 로그인 정확히 확인 |
| 0 | 끝내기 | 창 닫기 |

## 🧰 전체 기능 — RUN.bat 메뉴 (모든 기능)

`RUN.bat`은 모든 기능을 한 곳에 모은 메뉴입니다. 첫 화면은 **심플**, 분류 안에 **전부**:

- **자주 쓰는 것**: `1` 미리보기 배포 · `2` 진짜 공개 배포 · `3` 로그인 · `4` 내 계정 확인
- **분류별 더보기**: `5` 계정/팀 · `6` 프로젝트/배포 · `7` 환경 변수 · `8` 도메인/주소 · `9` 고급 · `10` 도구
- `11` **업데이트** (Vercel 최신 버전으로) · `0` 끝내기

## ⌨️ 명령어 표 (메뉴 ↔ 실제 Vercel 명령)

메뉴를 누르면 아래 실제 명령이 대신 실행됩니다. (직접 칠 필요 없음)

| 하는 일 | 실제 명령 |
|---|---|
| 미리보기 배포 | `vercel` |
| 진짜 공개 배포 | `vercel --prod` |
| 로컬에서 미리 실행 | `vercel dev` |
| 다시 배포 | `vercel redeploy` |
| 로그인 / 로그아웃 | `vercel login` / `vercel logout` |
| 내 계정 확인 | `vercel whoami` |
| 팀 목록 / 전환 | `vercel teams ls` / `vercel switch` |
| 폴더 연결 | `vercel link` |
| 배포 목록 / 상세 / 로그 | `vercel ls` / `vercel inspect <URL>` / `vercel logs <URL>` |
| 브라우저로 열기 | `vercel open` |
| 환경변수 목록/추가/삭제/받기 | `vercel env ls` / `add` / `rm` / `pull` |
| 도메인 목록/추가/삭제 | `vercel domains ls` / `add <도메인>` / `rm <도메인>` |
| DNS 기록 / 별칭 | `vercel dns ls <도메인>` / `vercel alias <URL> <도메인>` |
| 공개로 승격 / 되돌리기 | `vercel promote <URL>` / `vercel rollback <URL>` |
| 배포 삭제 / 설정 받기 | `vercel remove <URL>` / `vercel pull` |
| 새 프로젝트 / 도움말 / 버전 | `vercel init` / `vercel help` / `vercel --version` |
| 업데이트 | `npm install -g vercel@latest` |

## 🌐 미리보기 vs 진짜 공개 (중요)

| 구분 | 뜻 | 안전도 |
|---|---|---|
| **미리보기 배포** | 임시 주소로 올려 나만 확인 | 🟢 **안전(추천)** — 진짜 사이트는 그대로 |
| **진짜 공개 배포** | 방문자가 바로 보는 사이트로 올림 | 🔴 주의 — **`YES`(대소문자 무관)** 입력해야만 진행 |

> 실수로 눌러도 `YES`를 입력하기 전엔 절대 공개되지 않습니다.

## 🗑️ 제거 방법

- `시작하기.bat` → **`[3] 제거하기`** (또는 `UNINSTALL.bat` 더블클릭)
- **`YES`(대소문자 무관)** 입력해야만 지워집니다. **내가 만든 코드·파일은 지우지 않습니다.**
- 제거 대상: Vercel CLI(npm 전역 패키지) + 설정 폴더(`%USERPROFILE%\.vercel`).

## ❓ 문제 / 오류 대처

| 증상 | 이렇게 하세요 |
|---|---|
| "Node.js가 없습니다" | `[1] 설치하기` → **자동(Y)**, 또는 [nodejs.org](https://nodejs.org/)에서 **LTS** 받기 |
| 파란 SmartScreen 경고 | **추가 정보 → 실행** (새 파일에 뜨는 정상 경고) |
| 파일이 막힌 것 같아요 | `시작하기.bat`이 자동 차단 해제. 안 되면 우클릭 → 속성 → **차단 해제** |
| 설치가 자꾸 실패 | **백신 잠깐 끄고**(아래) 다시 / 와이파이 확인 / 다시 시도 |
| `vercel` 을 못 찾아요 | 창 닫고 **`시작하기.bat` 다시 실행**(새 창이라야 인식) |
| 영어 오류가 떴어요 | 이제 그 **아래에 한국어로 원인 + 다음 할 일**이 같이 떠요. 그대로 따라 하세요 |
| 로그인이 "확인 안 됨" | `[6]` → **`[4]` 로그인 정확히 확인** 으로 진짜 상태 확인 |
| 화살표가 안 먹혀요 | **번호키 + Enter** 로 고르면 됩니다 |
| 검은 영어 창이 무서워요 | 진행 중이란 뜻 — **닫지만 마세요** |

**💉 백신이 설치를 막을 때 (잠깐 끄는 법):**
1. 시작 → **Windows 보안** → 열기
2. **바이러스 및 위협 방지** → **설정 관리**
3. **실시간 보호**를 잠깐 **끔** → `[1] 설치하기` 다시
4. 설치가 끝나면 **반드시 다시 켜기** (중요!)

## 🔒 안전 / 개인정보

- 이 키트는 **관리자 권한 없이** 동작하며, 시스템 폴더를 건드리지 않습니다.
- 로그인 토큰은 Vercel 공식 CLI가 **내 사용자 폴더**에 저장합니다(키트가 따로 수집·전송하지 않음).
- 이 키트는 **외부로 어떤 데이터도 보내지 않습니다.** (Vercel 명령은 Vercel 서버와만 통신)
- 진짜 공개 배포·제거 같은 **되돌리기 어려운 작업은 `YES` 확인**을 받습니다.
